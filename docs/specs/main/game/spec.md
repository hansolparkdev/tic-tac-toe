# Spec: game

## Requirements

### Requirement: 3×3 보드 착수
사용자는 빈 셀을 클릭하거나 키보드로 선택해 현재 차례의 기호(X 또는 O)를 착수한다. 이미 놓인 셀은 무시되며 shake 피드백을 제공한다. 첫 판의 선공은 항상 X이다.

**시나리오: 빈 셀 착수**
- Given: 초기 빈 보드, 현재 차례 X
- When: 사용자가 (1,1) 셀을 클릭한다
- Then: (1,1)에 X가 렌더링되고 현재 차례가 O로 바뀐다

**시나리오: 점유 셀 클릭 무시**
- Given: (1,1)에 X가 놓여 있고 현재 차례는 O
- When: 사용자가 (1,1)을 다시 클릭한다
- Then: 보드 상태는 변하지 않고 해당 셀에 shake 애니메이션이 재생된다 (motion-reduce 시 생략)

### Requirement: 승패/무승부 판정
가로·세로·대각선 중 하나라도 같은 기호 3개가 연속되면 해당 기호의 승리로 즉시 선언한다. 9칸이 모두 찼는데 승자가 없으면 무승부이다. 승리 시 승리 라인 3칸을 관통하는 SVG 오버레이가 그려진다.

**시나리오: 가로 승리**
- Given: X가 (0,0)(0,1)에 놓였고 O가 (1,0)(1,1)에 놓인 상태, 차례 X
- When: X가 (0,2)에 착수한다
- Then: 상태 메시지에 "X 승리!"가 표시되고, 승리 라인이 (0,0)-(0,2) 중심을 관통해 그려지며, 이후 셀 클릭은 무시된다

**시나리오: 무승부**
- Given: 9칸 중 8칸이 채워졌고 어느 라인도 3연속이 아님, 마지막 셀 남음
- When: 마지막 셀에 착수된다
- Then: 상태 메시지에 "무승부!"가 표시되고 winningLine은 null이다

### Requirement: 승리 라인 반응형 렌더링
승리 라인은 SVG로 보드 좌표 기반 렌더링된다. 보드 크기가 변경되면 셀 중심 좌표를 재계산해 라인이 항상 셀 중앙을 관통하도록 다시 그린다.

**시나리오: 리사이즈**
- Given: 승리 상태에서 라인이 렌더링되어 있음
- When: 윈도우 너비가 변해 보드 크기가 달라진다
- Then: ResizeObserver 콜백으로 셀 좌표가 재계산되어 라인이 새로운 중심을 관통한다

### Requirement: 라운드 리셋과 선공 교대
"다시하기" 버튼은 보드만 초기화한다. 다음 라운드의 선공은 직전 라운드의 패자이며, 직전 라운드가 무승부면 직전 선공의 반대 기호가 선공이다. 스코어는 이 슬라이스 범위 밖이다.

**시나리오: 패자 선공**
- Given: X가 승리한 직후 상태
- When: 사용자가 "다시하기"를 누른다
- Then: 보드가 비워지고 현재 차례는 O(직전 패자)이다

**시나리오: 무승부 후 선공**
- Given: 직전 라운드가 무승부이고 해당 라운드 선공은 X였다
- When: 사용자가 "다시하기"를 누른다
- Then: 보드가 비워지고 현재 차례는 O이다

### Requirement: 종료된 보드 보호
승리 또는 무승부 이후 셀 클릭은 보드 상태를 변경하지 않으며, "다시하기" 버튼에 pulse 피드백을 제공한다.

**시나리오: 종료 후 클릭**
- Given: X 승리 상태
- When: 사용자가 빈 셀을 클릭한다
- Then: 보드는 변하지 않고 "다시하기" 버튼에 pulse 애니메이션이 재생된다 (motion-reduce 시 생략)

### Requirement: 키보드 네비게이션
사용자는 마우스 없이 모든 게임플레이를 수행할 수 있다. Tab으로 보드에 진입하고, Arrow(↑↓←→)로 셀 간 이동, Enter 또는 Space로 착수, Tab으로 다시하기 버튼에 이동한다. 포커스된 셀은 2px 포커스 링으로 표시된다.

**시나리오: 화살표 이동 + Enter 착수**
- Given: 초기 빈 보드, 보드에 포커스 진입하여 (0,0)이 포커스됨
- When: Arrow Right 후 Enter를 누른다
- Then: 포커스가 (0,1)로 이동하고 (0,1)에 X가 착수된다

### Requirement: 접근성 aria 속성
각 셀은 위치와 상태를 담은 `aria-label`을 가진다(예: "2행 3열, 비어있음"/"2행 3열, X"). 상태 메시지 영역은 `aria-live="polite"`로 승패 결과를 스크린리더에 전달한다.

**시나리오: 착수 시 aria-label 갱신**
- Given: (1,1) 셀의 aria-label이 "2행 2열, 비어있음"
- When: X가 (1,1)에 착수된다
- Then: 해당 셀의 aria-label이 "2행 2열, X"로 갱신된다

### Requirement: prefers-reduced-motion 대응
사용자가 OS 수준에서 모션 감소를 선호하면 pop, shake, pulse, 승리 라인 draw 애니메이션이 비활성화되거나 0.05초 이하로 축소된다. 상태 전환 자체는 유지된다.

**시나리오: 모션 감소 시 승리 라인**
- Given: `prefers-reduced-motion: reduce`가 설정된 환경
- When: 승리가 확정되어 라인이 표시된다
- Then: draw 애니메이션 없이 즉시(또는 0.05초 이내) 라인이 최종 상태로 렌더링된다

### Requirement: 누적 스코어보드
헤더에 X 승수 / 무승부 횟수 / O 승수를 보여주는 가로 3분할 카드가 표시된다. 현재 차례에 해당하는 기호(X 또는 O) 카드에 강조 테두리가 적용된다.

**시나리오: 초기 표시**
- Given: 저장된 스코어가 없는 첫 방문
- When: 페이지가 로드된다
- Then: 스코어보드는 "X: 0승 / Draw: 0 / O: 0승"을 표시하고, X 카드에 강조 테두리가 있다

**시나리오: 차례에 따른 강조 이동**
- Given: 현재 차례 X인 진행 상태
- When: X가 한 수를 둔다
- Then: 강조 테두리가 O 카드로 이동한다

### Requirement: 승리/무승부 시 스코어 갱신
라운드가 승리 또는 무승부로 종료되면 해당 항목의 숫자가 1 증가하고 카운트업 애니메이션이 재생된다. `prefers-reduced-motion` 시 애니메이션 없이 즉시 값이 반영된다.

**시나리오: X 승리 시 X 점수 증가**
- Given: 스코어가 X=1, draw=0, O=0이고 X가 승리 라인을 완성하는 수를 두기 직전
- When: X가 승리 수를 착수한다
- Then: X 스코어가 2로 증가하며 카운트업 애니메이션이 재생된다

**시나리오: 무승부 시 draw 증가**
- Given: 8칸이 채워졌고 승자 없이 마지막 빈 칸만 남은 상태
- When: 마지막 칸이 착수되어 무승부가 된다
- Then: draw 스코어가 1 증가한다

### Requirement: 스코어 리셋 확인 다이얼로그
"스코어 리셋" 버튼 클릭 시 확인 다이얼로그가 나타난다. 다이얼로그는 헤더 "스코어 리셋", 본문 "스코어를 0부터 다시 시작할까요?", [취소]/[리셋] 버튼을 포함한다. 기본 포커스는 [취소]이며, 다이얼로그 내부에서만 Tab 포커스가 순환한다.

**시나리오: 기본 포커스와 ESC 취소**
- Given: 게임 진행 중 스코어 X=2, O=1
- When: "스코어 리셋"을 클릭한 뒤 ESC를 누른다
- Then: 다이얼로그 오픈 시 [취소] 버튼에 포커스가 있고, ESC 후 다이얼로그가 닫히며 스코어는 X=2, O=1로 그대로 유지된다

**시나리오: 오버레이 클릭 취소**
- Given: 다이얼로그가 열려 있음
- When: 사용자가 오버레이(모달 외부 영역)를 클릭한다
- Then: 다이얼로그가 닫히고 스코어는 변경되지 않는다

**시나리오: [리셋] 확정**
- Given: 스코어 X=2, draw=1, O=1이고 다이얼로그가 열려 있음
- When: 사용자가 [리셋]을 클릭한다
- Then: 스코어가 0/0/0으로 초기화되고, 보드가 비워지며, 다음 라운드 선공이 X로 리셋되고, 다이얼로그가 닫힌다

**시나리오: 포커스 트랩**
- Given: 다이얼로그가 열려 있고 [취소]에 포커스
- When: Tab을 반복해서 누른다
- Then: 포커스는 [취소] ↔ [리셋] 사이에서만 순환하며 다이얼로그 외부로 나가지 않는다

### Requirement: sessionStorage 스코어/선공 영속화
누적 스코어와 다음 라운드 선공은 sessionStorage 키 `tic-tac-toe/v1`에 저장된다. 새로고침 시 값이 복원되며, 진행 중이던 보드는 복원하지 않고 빈 보드로 시작한다. 탭이 종료되면 sessionStorage 정책에 따라 자동으로 초기화된다.

**시나리오: 새로고침 후 복원**
- Given: 스코어 X=2, O=1, draw=0, 다음 선공 O인 상태에서 O가 한 수 착수한 진행 중 보드
- When: 사용자가 브라우저를 새로고침한다
- Then: 스코어는 X=2, O=1, draw=0으로 복원되고, 현재 차례는 O(= nextStarter)이며, 보드는 빈 상태이다

**시나리오: 저장 포맷 손상 시 기본값**
- Given: sessionStorage `tic-tac-toe/v1` 값이 파싱 불가능한 문자열로 설정되어 있음
- When: 앱이 마운트된다
- Then: 기본값(scores 0/0/0, nextStarter='X')으로 시작하고 예외는 사용자에게 노출되지 않는다

**시나리오: 탭 종료 후 재오픈**
- Given: 스코어 X=3인 상태로 탭을 닫았다
- When: 사용자가 동일 URL을 새 탭에서 연다
- Then: 스코어는 0/0/0, 선공은 X로 초기화되어 시작한다

### Requirement: 스코어보드 접근성
스코어 카드는 스크린리더가 의미를 파악할 수 있도록 기호·값·단위를 포함한 접근 가능한 텍스트(aria-label 또는 가시 텍스트)를 가진다. 스코어 갱신은 상태 메시지와 별개로 시각적 강조로만 전달되어 aria-live 스팸을 유발하지 않는다.

**시나리오: 카드 접근 가능 텍스트**
- Given: 스코어 X=2, draw=1, O=0
- When: 스크린리더가 X 카드를 읽는다
- Then: "X 2승"에 상응하는 접근 가능한 텍스트가 전달된다
