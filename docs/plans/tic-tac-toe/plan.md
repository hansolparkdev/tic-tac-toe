# 기획서: Tic-Tac-Toe (틱택토) 웹 게임

## 1. 배경 및 목적
- **왜 이 기능이 필요한가**: 짧은 휴식 시간에 가볍게 즐길 수 있는 2인용 클래식 게임을 웹에서 빠르게 제공하기 위함
- **해결하려는 문제**: 설치/회원가입 없이 즉시 시작할 수 있는 심플한 대전 게임 부재
- **기대 효과 및 성공 지표**:
  - E2E 테스트에서 보드 렌더 후 **100ms 내 첫 셀 클릭이 가능**함을 검증
  - 수동 QA 체크리스트로 **2라운드 연속 플레이 흐름**(1라운드 종료 → 스코어 반영 → 2라운드 선공 교대 → 진행)이 끊김 없이 수행되는지 검증
  - Lighthouse Accessibility 점수 95점 이상
- **의도된 단순화**: 첫 판의 선공은 항상 **X 고정**이며, 선공 선택 UI는 제공하지 않는다(MVP 범위)

## 2. 사용자
- **주요 사용자**: 10대~40대 일반 웹 사용자, 짧은 여가 시간을 활용하려는 2인
- **사용 맥락**: 같은 기기(데스크탑/태블릿/모바일) 앞에 두 사람이 번갈아 착수하는 오프라인 대면 플레이
- **사용자 목표**:
  - 회원가입/로그인 없이 즉시 플레이
  - 누적 스코어를 통해 여러 판 연속 대결
  - 직관적이고 명확한 승패 인식

## 3. 핵심 기능
- **기능 1 — 3×3 보드 플레이**: 셀을 클릭해 X/O를 번갈아 착수. 이미 놓인 셀은 비활성화
- **기능 2 — 승패/무승부 판정**: 가로·세로·대각선 3연속 일치 시 승리 즉시 선언. 9칸 모두 채우면 무승부
- **기능 3 — 누적 스코어보드**: X 승수, O 승수, 무승부 횟수를 상단에 표시
- **기능 4 — 라운드 리셋 & 전체 리셋**: "다시하기"는 보드만 초기화, "스코어 리셋"은 스코어까지 0으로 초기화
- **기능 5 — 선공 교대 규칙**: 라운드 종료 시 **직전 패자가 다음 라운드 선공**, 무승부 시 **직전 선공의 반대 기호**가 선공
- **기능 6 — 스코어 영속화 (sessionStorage)**: 브라우저 새로고침 시에도 현재 세션 동안 누적 스코어와 다음 라운드 선공 정보가 유지됨. 탭 종료 시에는 초기화

## 4. 사용자 흐름 (User Flow)

**정상 흐름:**
1. 사용자가 페이지에 접속한다
2. 시스템이 빈 3×3 보드와 스코어 0:0:0, 현재 차례 "X"를 표시한다
3. 사용자 A가 임의의 빈 셀을 클릭한다
4. 시스템이 해당 셀에 X를 렌더링하고 차례를 O로 전환한다
5. 사용자 B가 다른 빈 셀을 클릭한다
6. 시스템이 O를 렌더링하고 승패를 판정한다
7. 3연속이 완성되면 시스템이 승리 라인을 하이라이트하고, 스코어를 갱신하며, "X 승리!" 같은 메시지를 표시한다
8. 사용자가 "다시하기" 버튼을 누르면 보드만 초기화되고 다음 라운드 선공(직전 패자) 기호로 차례가 세팅된다
9. 무승부 시 다음 라운드 선공은 직전 선공의 반대 기호로 세팅된다

**예외 흐름:**
- **이미 놓인 셀 클릭**: 사용자가 점유된 셀 클릭 → 시스템이 무시하고 셀을 살짝 흔드는 피드백(shake) 제공
- **게임 종료 후 셀 클릭**: 사용자가 종료된 보드의 셀 클릭 → 시스템이 무시하고 "다시하기" 버튼을 강조(pulse)
- **스코어 리셋 확인**: 사용자가 "스코어 리셋" 클릭 → 시스템이 확인 다이얼로그 표시 → "리셋" 선택 시 모든 스코어 0으로 초기화, "취소" 또는 ESC 시 변경 없음
- **새로고침**: 사용자가 플레이 중 새로고침 → 시스템이 sessionStorage에서 스코어와 다음 선공 정보 복원, 진행 중이던 보드는 초기 상태로 시작

## 5. 화면 구성

### 화면 1: 메인 게임 화면 (단일 페이지)
- **목적**: 모든 플레이가 이 한 화면에서 이루어짐
- **레이아웃**: 세로 중앙 정렬 단일 컬럼 구조
  - 상단: 헤더 (타이틀 + 스코어보드)
  - 중앙: 게임 보드
  - 하단: 상태 메시지 + 액션 버튼
- **주요 요소**:
  - **타이틀**: 최상단, "Tic-Tac-Toe", 28~32px
  - **스코어보드**: 타이틀 하단, 가로 3분할 카드 (X: n승 / Draw: n / O: n승). 현재 차례의 기호 카드에 강조 테두리
  - **현재 차례 인디케이터**: 스코어보드와 보드 사이, "다음 차례: X" 텍스트와 기호 아이콘
  - **게임 보드**: 화면 중앙, 3×3 정사각 그리드. 한 변 최대 360px (모바일에서는 화면 너비의 80%)
    - 각 셀: 정사각형, 굵은 셀 경계선(2px), 호버 시 배경색 변화
    - 착수된 기호: 셀 중앙 정렬, 셀 크기의 60% 글자 크기. X는 브랜드 블루, O는 브랜드 코랄
  - **승리 라인 오버레이**: 승리 셀 3개를 관통하는 라인. **SVG로 보드 좌표 기반 렌더링**하며, 화면 리사이즈 시 셀 좌표를 재계산해 라인이 정확히 겹치도록 다시 그림. 0.3초 draw 애니메이션
  - **상태 메시지**: 보드 하단, "X 승리!" / "무승부!" / "O 차례입니다" 등. 높이 고정으로 레이아웃 점프 방지
  - **액션 버튼 영역**: 가로 2개 버튼
    - 좌: "다시하기" (Primary, 보드만 리셋)
    - 우: "스코어 리셋" (Secondary, 다이얼로그 후 전체 리셋)
- **인터랙션**:
  - 셀 클릭/탭 → 기호 착수, 0.15초 pop 애니메이션
  - 점유 셀 클릭 → shake 애니메이션
  - 승리 확정 → 승리 라인 draw + 스코어 카드 숫자 카운트업
  - **키보드 보드 네비게이션**: 보드 포커스 상태에서 **Arrow keys(↑↓←→)로 셀 간 이동**, **Enter 또는 Space로 착수**. 포커스된 셀은 2px 포커스 링으로 표시
  - 다시하기 버튼: 0.2초 fade 후 보드 재생성
- **상태**:
  - **초기**: 빈 보드, "다음 차례: X"
  - **진행 중**: 일부 셀 착수됨
  - **승리**: 승리 라인 표시, 셀 클릭 비활성, 버튼 강조
  - **무승부**: 모든 셀 회색톤, 버튼 강조
  - **로딩**: sessionStorage 복원 시 짧은 skeleton (100ms 이내)

### 화면 2: 스코어 리셋 확인 다이얼로그 (모달)
- **목적**: 실수로 인한 스코어 초기화 방지
- **레이아웃**: 화면 중앙 모달, 오버레이 (반투명 검정 50%)
- **주요 요소**:
  - **헤더**: "스코어 리셋"
  - **본문 문구**: "스코어를 0부터 다시 시작할까요?"
  - **버튼 영역** (가로 2개, 우측 정렬):
    - [취소] (Secondary) — **기본 포커스**
    - [리셋] (Destructive, 적색 계열)
- **인터랙션**:
  - [취소] 또는 **ESC 키** → 다이얼로그 닫기, 변경 없음
  - [리셋] → 스코어 0:0:0 초기화, 보드 초기화, 다음 선공 X로 리셋, 다이얼로그 닫기
  - 오버레이 클릭 → [취소]와 동일
  - Tab 포커스 트랩: 모달 내부에서만 포커스 순환
- **상태**: 열림/닫힘 단일 상태, 0.15초 fade-in/out

## 6. UI/UX 원칙
- **톤앤매너**: 모던, 미니멀, 장난기 있는 친근함. 불필요한 장식 제거
- **컬러 방향**:
  - 배경: 화이트 또는 라이트 그레이(#F7F8FA)
  - X: 블루(#3B82F6), O: 코랄(#F97316), 승리 라인: 기호 색상과 동일
  - Primary 버튼: 다크 네이비, Destructive: 레드(#EF4444)
  - 다크모드는 MVP에서 제외 (미결 사항 참조)
- **핵심 UX 원칙**:
  - **3클릭 이내 1라운드 완료 후 재시작 가능** (착수 제외)
  - 모든 상태 전환에 시각적 피드백 (pop, shake, fade, draw)
  - 오류 상황 없음 — 잘못된 행동은 조용히 무시하되 힌트 제공
- **접근성**:
  - 모든 셀에 aria-label (예: "1행 2열, 비어있음" / "2행 2열, X")
  - 상태 메시지는 aria-live="polite"로 스크린리더 읽기
  - 키보드만으로 모든 기능 조작 가능 (Tab, Arrow, Enter, Space, ESC)
  - **prefers-reduced-motion**: 사용자가 모션 감소 설정 시 모든 애니메이션(shake, fade, pop, 승리 라인 draw, 카운트업)을 **비활성화하거나 0.05초 이하로 축소**. 상태 전환은 유지하되 움직임만 제거
  - 색상만으로 정보 전달하지 않음 (X/O 기호 자체로 구분)

## 7. 제약 조건
- **기술적 제약**:
  - React 18 + TypeScript + Vite + Tailwind CSS
  - `any` 타입 금지, `console.log` 금지
  - 함수형 컴포넌트만 사용, Props 타입 명시 필수
  - 단위 테스트는 Vitest + RTL, E2E는 Playwright
  - 서버/백엔드 없음 (프론트 단독)
- **비즈니스 제약**:
  - 회원 기능/서버 저장 없음
  - 광고 없음
  - 2인 로컬 플레이만 지원 (AI/온라인 대전 제외)
- **기간/리소스 제약**: MVP 범위로 단일 스프린트 내 완료
